System Background

The Logistics Tracking System (LTS) is a cloud-based platform that tracks high-value assets in transit. The system processes Controlled Unclassified Information (CUI) including shipment manifests, customer data, and real-time asset location tracking.

System Location: AWS GovCloud (us-gov-west-1)
Impact Level: Moderate
Users: Logistics coordinators, security officers, program managers
System Boundary: EC2 instances, S3 buckets, RDS databases, VPC with public/private subnets

Trucking Connection: This system mirrors the paper-based compliance systems I managed for 13 years—now digitized with security controls built in.

Implementation Statement Example

AC-3: Access Enforcement

The Logistics Tracking System enforces access control policies through AWS IAM with least privilege principles. All users are assigned roles with permissions scoped to their job functions. MFA is enforced for all console users. IAM policies deny access to resources outside the user's authorized scope. Access reviews are conducted quarterly with evidence retained in AWS Config.

POA&M Sample

Finding: AC-3 Partially Implemented (Legacy Service Accounts)

Three legacy service accounts lack MFA enforcement, creating a potential vector for unauthorized CUI access.

Remediation Plan:

  • Migrate legacy accounts to IAM roles with least privilege
  • Enforce MFA for all service accounts by May 30, 2026
  • Implement weekly credential audit

Milestone Date: Q2 2026 · Status: In progress · Resources Required: DevOps support

Executive Summary

"The Logistics Tracking System is a moderate-impact system handling CUI. This RMF package documents five key controls—AC-3, SC-7, AU-2, SI-4, and IA-2—selected for their critical role in protecting CUI in transit. Implementation leverages AWS native services including IAM, security groups, CloudTrail, and GuardDuty. One POA&M item is documented for legacy account remediation, with completion targeted for Q2 2026. With these controls implemented, the system is positioned for an ATO and CMMC Level 2 assessment."

CMMC Applicability

The controls selected map directly to NIST 800-171 requirements, making this system ready for CMMC Level 2 assessment with proper evidence collection:

  • AC-3 → AC.1.001 (Limit information system access), AC.2.007 (Employ least privilege)
  • SC-7 → SC.2.064 (Boundary protection)
  • AU-2 → AU.2.041 (Audit events), AU.2.042 (Audit log content)
  • SI-4 → SI.2.043 (System monitoring)
  • IA-2 → IA.2.006 (Identification), IA.2.008 (MFA)

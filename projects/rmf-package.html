RMF Package · Logistics Tracking System (LTS)

Overview

A complete RMF (Risk Management Framework) package for a fictional Logistics Tracking System. The system tracks high-value assets in transit—directly reflecting my 13 years of experience managing assets under federal safety regulations.

Purpose: Demonstrate understanding of NIST 800-53 controls, control implementation statements, POA&M management, and CMMC applicability.

Artifacts Included

  • System Description: Logistics Tracking System (LTS) with trucking background tie-in
  • 5 NIST 800-53 Controls: AC-3 (Access Enforcement), SC-7 (Boundary Protection), AU-2 (Audit Events), SI-4 (System Monitoring), IA-2 (Identification & Authentication)
  • Control Implementation Statements: Each control documented with specific AWS service implementations
  • Control Selection Rationale: Why these five controls were selected for a moderate-impact system
  • CMMC Applicability Section: Mapping each control to NIST 800-171 / CMMC Level 2 practices
  • POA&M: One fictional finding with remediation plan and milestone dates
  • Executive Summary: 5-sentence summary for non-technical leadership

Control Implementation Example

Control: AC-3 (Access Enforcement)

Implementation Statement: The Logistics Tracking System enforces access control policies through AWS IAM with least privilege principles. All users are assigned roles with permissions scoped to their job functions. MFA is enforced for all console users. IAM policies deny access to resources outside the user's authorized scope.

CMMC Mapping: AC.1.001, AC.2.007, AC.2.013

Frameworks & Controls

NIST 800-53: AC-3, SC-7, AU-2, SI-4, IA-2

CMMC Level 2 / NIST 800-171: 14 control families with specific mappings for each selected control

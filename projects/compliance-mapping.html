NIST 800-53 to HIPAA Security Rule Mapping

In Progress

Project Overview

This project maps the five NIST 800-53 controls from my RMF Authorization Package to the HIPAA Security Rule. The goal is to demonstrate that I can translate controls across frameworks, which is a core skill for GRC roles in healthcare and civilian sectors.

Healthcare organizations in Nashville and across the country operate under HIPAA. Showing that I can map NIST controls to HIPAA requirements proves that my RMF experience is directly applicable to their environment.

Why This Mapping Matters

  • NIST 800-53 is the standard for federal and defense systems.
  • HIPAA Security Rule is the standard for healthcare.
  • Both frameworks require access control, audit logging, and monitoring.
  • The mapping shows that I understand the underlying security objectives, not just the checklist items.

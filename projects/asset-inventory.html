Asset Inventory + CUI Data Flow

Overview

A complete asset inventory for a mock logistics company, designed to support CMMC scoping requirements. The inventory tracks 20 assets with CUI designation, location, and protection status. A data flow diagram shows how CUI moves through the environment.

Purpose: Demonstrate ability to track assets, scope CUI, and document data flows—critical skills for CMMC assessments.

Artifacts Included

  • Asset Inventory Spreadsheet: 20 assets (servers, workstations, network devices, cloud services, SaaS applications)
  • CUI Designation: Each asset marked as "CUI flows through," "CUI stored/processed," or "Not applicable"
  • Asset Location: On-prem, AWS region, or SaaS provider
  • Protection Status: Encryption, MFA, patching schedule, logging status
  • CUI Data Flow Diagram: Visual representation of how CUI enters, moves through, and exits the system
  • Methodology Document: Scoping criteria and inventory management approach

Asset Example

Asset: LTS Application Server (EC2)
Location: AWS GovCloud (us-gov-west-1)
CUI Status: Stores and processes CUI (shipment manifests, customer data)
Protection: Encrypted EBS, MFA-enabled IAM roles, weekly patching, CloudTrail logging
Owner: IT Operations

Framework

CMMC Level 2: Asset Management domain (AM.1.001, AM.2.002, AM.2.003) — requirement to identify, inventory, and protect assets handling CUI.

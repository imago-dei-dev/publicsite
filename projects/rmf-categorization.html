RMF in Practice: System Categorization

Status: In Progress – Expected completion: March 2025

RMF Phases: Categorize, Select

Relevant Roles: ISSO, RMF Analyst, Security Assessor

Project Overview

This project demonstrates understanding of the first two phases of the Risk Management Framework (RMF): Categorize and Select. Using a mock system ("HR Database"), I document the system's purpose, data types, and impact levels following NIST guidance, then select appropriate security controls from NIST SP 800-53.

Tools Used

  • NIST SP 800-60 – Impact level determination guidance
  • NIST SP 800-53 Rev. 5 – Security control catalog
  • FIPS 199 – Security categorization standards
  • Draw.io – System boundary diagram

System Description: HR Database

System Name: Human Resources Management System (HRMS)

Purpose: Store and process employee records, payroll information, and performance reviews

Users: HR staff (20), managers (50), employees (500) – self-service view only

Data Types:

  • Personally Identifiable Information (PII): names, SSNs, addresses, birth dates
  • Financial data: salary, bank account details for direct deposit
  • Personnel records: performance reviews, disciplinary actions

System Boundary Diagram

The system boundary includes:

  • Web servers (public-facing for employee self-service)
  • Application servers (HR workflows)
  • Database servers (employee data)
  • Authentication services (Active Directory integration)
  • Backup systems and offsite storage

Step 1: Security Categorization (FIPS 199)

Following FIPS 199 and NIST SP 800-60, I determined impact levels for each security objective:

Confidentiality: Moderate

The system contains PII and financial data. Unauthorized disclosure would cause serious harm to employees and legal liability for the organization.

Integrity: Moderate

Modification of payroll or personnel data could result in financial loss, incorrect payments, or legal consequences.

Availability: Low

While the system is important, short-term outages (24-48 hours) would cause inconvenience but not life-threatening or mission-critical impact. Payroll can be processed manually if needed.

Overall System Impact Level: MODERATE (highest of any category)

Step 2: Control Selection (NIST SP 800-53)

Based on the Moderate impact level, I selected the appropriate control baseline from NIST SP 800-53 Rev. 5. Below are key controls by family:

AC (Access Control):

  • AC-2: Account Management – Document account provisioning/deprovisioning
  • AC-3: Access Enforcement – Role-based access control implementation
  • AC-6: Least Privilege – Restrict HR staff to minimum necessary permissions
  • AC-7: Unsuccessful Logon Attempts – Account lockout after 5 failed attempts

AU (Audit and Accountability):

  • AU-2: Audit Events – Log all access to PII, privilege changes
  • AU-6: Audit Review – Weekly log review by security team
  • AU-12: Audit Generation – Configure systems to generate required logs

SC (System and Communications Protection):

  • SC-8: Transmission Confidentiality – TLS for all web traffic
  • SC-12: Cryptographic Key Management – Use AWS KMS for key management
  • SC-13: Cryptographic Protection – FIPS 140-2 validated cryptography
  • SC-28: Protection of Information at Rest – Encrypt database volumes

SI (System and Information Integrity):

  • SI-2: Flaw Remediation – Monthly patching cycle
  • SI-4: System Monitoring – Wazuh SIEM integration
  • SI-7: Software Integrity – Verify file integrity monthly

Control Implementation Summary

For each selected control, I documented:

  • Responsible party: System Owner, ISSO, or Cloud Provider
  • Implementation status: Planned, Partially Implemented, or Implemented
  • Evidence source: System documentation, configuration files, or scan results

Key Takeaways

  • System categorization drives all subsequent RMF steps
  • Impact levels determine which controls apply
  • Documentation must be clear and repeatable for assessment
  • Control selection balances security requirements with operational needs
  • This project directly supports RMF Steps 1 (Categorize) and 2 (Select)

Vulnerability Management · Nessus Scan Summary

Overview

A lightweight vulnerability scan against a vulnerable target (Metasploitable). The summary identifies the top three findings by CVSS score, documents remediation plans, and maps each to NIST controls.

Purpose: Demonstrate ability to run scans, interpret findings, prioritize by risk, and translate technical vulnerabilities into actionable remediation plans.

Artifacts Included

  • Scan Summary: One-page executive summary with top findings
  • Top 3 Findings: Each with CVSS score, vulnerability description, and risk justification
  • Remediation Plans: Step-by-step remediation for each finding
  • Control Mappings: Each finding mapped to NIST 800-53 controls
  • Screenshots: Evidence of scan execution and findings

Finding Example

Finding 1: Apache Tomcat Default Credentials (CVSS 9.8)

Remediation: Remove default accounts immediately. Implement strong password policy for all administrative accounts. Restrict Tomcat manager access to trusted IP ranges.
NIST Mapping: CM-6 (Configuration Settings), IA-2 (Identification & Authentication)

Frameworks & Controls

NIST 800-53: RA-5 (Vulnerability Scanning), SI-4 (System Monitoring), CM-6 (Configuration Settings), IA-2 (Identification & Authentication)

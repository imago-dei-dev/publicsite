System Background

The Logistics Tracking System (LTS) is a cloud-based platform that tracks high-value assets in transit. The system processes Controlled Unclassified Information (CUI) including shipment manifests, customer data, and real-time asset location tracking.

System Location: AWS GovCloud (us-gov-west-1)
Impact Level: Moderate
Users: Logistics coordinators, security officers, program managers
System Boundary: EC2 instances, S3 buckets, RDS databases, VPC with public/private subnets

Trucking Connection: This system mirrors the paper-based compliance systems I managed for 13 years—now digitized with security controls built in.

Executive Summary

"The Logistics Tracking System is a moderate-impact system handling CUI. This RMF package documents five key controls—AC-3, SC-7, AU-2, SI-4, and IA-2—selected for their critical role in protecting CUI in transit. Implementation leverages AWS native services including IAM, security groups, CloudTrail, and GuardDuty. Five POA&M items are documented with remediation plans and target completion dates through June 2026. With these controls implemented, the system is positioned for authorization."

POA&M Summary

The POA&M identifies five findings across the selected controls:

  • IA-2: Contractors lack unique credentials (Medium risk) — target 05/15/2026
  • AC-3: Legacy service accounts lack MFA (High risk) — target 05/30/2026
  • SC-7: No IDS at API Gateway (Medium risk) — target 06/30/2026
  • SI-4: GuardDuty alerts not configured (High risk) — target 06/15/2026
  • AU-2: Audit logs not reviewed regularly (Low risk) — target 05/01/2026

Each finding includes a remediation plan, responsible party, and milestone dates. Full details in the downloadable PDF.

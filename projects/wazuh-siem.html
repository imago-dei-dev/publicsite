Wazuh SIEM: Security Monitoring Lab

Status: In Progress – Expected completion: March 2025

RMF Phase: Monitor

Relevant Roles: SOC Analyst, Security Monitoring, ISSO

Project Overview

This project demonstrates implementation of a Security Information and Event Management (SIEM) system using Wazuh, an open-source SIEM platform. The lab includes a Wazuh server, multiple agents (Windows and Linux), custom alert rules, and a security dashboard for continuous monitoring—directly supporting RMF Step 6 (Monitor).

Tools Used

  • Wazuh Server – SIEM platform (Ubuntu 22.04)
  • Wazuh Agents – Windows 10, Ubuntu, Kali Linux
  • Elastic Stack – Kibana for visualization
  • VirtualBox – Lab environment

Step 1: Lab Architecture

The lab consists of:

  • Wazuh Server (Ubuntu 22.04): 4GB RAM, 2 vCPUs, 50GB storage
  • Windows 10 Agent: Simulating a typical workstation
  • Ubuntu Server Agent: Simulating a Linux server
  • Kali Linux Agent: Used for attack simulation

All VMs are on an isolated host-only network.

Step 2: Wazuh Server Installation

I installed Wazuh using the all-in-one deployment script: 

curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh --generate-config-files

The installation includes the Wazuh indexer, server, and dashboard components.

Step 3: Agent Deployment

I deployed agents on each target system:

Windows agent installation:

# Download and install Wazuh agent for Windows
# Configure manager IP: 192.168.56.10
# Start the agent service

Linux agent installation:

curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo apt-key add -
echo "deb https://packages.wazuh.com/4.x/apt/ stable main" | sudo tee /etc/apt/sources.list.d/wazuh.list
sudo apt update
sudo apt install wazuh-agent
sudo systemctl start wazuh-agent

Step 4: Custom Alert Rules

I created custom alert rules for security-relevant events: 

<group name="windows,authentication_failures,">
  <rule id="100001" level="10">
    <if_sid>60106</if_sid>
    <match>An account failed to log on</match>
    <description>Windows: Multiple failed login attempts detected</description>
    <group>authentication_failure,multiple_failures,</group>
  </rule>
</group>

These rules trigger alerts for:

  • Multiple failed login attempts (brute force detection)
  • Service installation or modification
  • New user account creation
  • Firewall rule changes

Step 5: Security Dashboard

I configured Kibana dashboards to visualize:

  • Security events over time
  • Alert severity distribution
  • Top affected agents
  • Compliance scorecard (PCI DSS, NIST 800-53 mappings)

Note: Dashboard screenshots will be added when connectivity improves.

Step 6: Attack Simulation

Using the Kali Linux agent, I simulated attacks to trigger alerts:

  • Brute force simulation: Multiple failed SSH logins to Ubuntu server
  • Malware simulation: Created test files with EICAR pattern
  • Privilege escalation attempt: Simulated using Metasploit

All simulations successfully triggered alerts in the Wazuh dashboard.

Key Takeaways

  • SIEM deployment provides real-time visibility into security events
  • Custom alert rules reduce false positives and focus on relevant threats
  • Agent-based monitoring enables detailed endpoint visibility
  • Dashboards communicate security posture to stakeholders
  • This lab directly supports RMF Step 6 (Continuous Monitoring)
